# 智慧城市边缘视频分析平台 - 部署文档

## 一、环境要求

### 1.1 硬件要求
- CPU: 4核及以上 (推荐8核)
- 内存: 8GB及以上 (推荐16GB)
- 磁盘: 50GB可用空间
- GPU: 可选 (用于YOLOv8加速)

### 1.2 软件要求
- Docker 20.10+
- Docker Compose 2.0+
- 或: Java 11+, Maven 3.6+, MySQL 8.0+, Redis 7.0+

### 1.3 端口要求
| 服务 | 端口 | 说明 |
|------|------|------|
| MySQL | 3306 | 数据库 |
| Redis | 6379 | 缓存 |
| YOLOv8 | 5000 | AI推理服务 |
| 后端 | 8080 | Spring Boot API |

## 二、Docker Compose 部署 (推荐)

### 2.1 环境准备

```bash
# 克隆项目
git clone <repository-url>
cd EdgeVideoAnalysis

# 可选: 修改默认密码 (编辑 .env 文件)
cat > .env << EOF
MYSQL_ROOT_PASSWORD=your_mysql_password
REDIS_PASSWORD=your_redis_password
MYSQL_DATABASE=edge_video_analysis
EOF
```

### 2.2 构建与启动

```bash
# 构建并启动所有服务
docker-compose up -d --build

# 查看服务状态
docker-compose ps

# 查看日志
docker-compose logs -f
```

### 2.3 服务健康检查

```bash
# 检查后端健康
curl http://localhost:8080/api/health

# 检查YOLOv8服务
curl http://localhost:5000/health

# 检查MySQL
mysql -h127.0.0.1 -P3306 -uroot -p123456 -e "SELECT 1"

# 检查Redis
redis-cli -h 127.0.0.1 -p 6379 -a 123456 ping
```

### 2.4 停止与清理

```bash
# 停止服务
docker-compose down

# 停止并删除数据卷
docker-compose down -v

# 重启服务
docker-compose restart
```

## 三、本地开发部署

### 3.1 数据库初始化

```sql
-- 创建数据库
CREATE DATABASE IF NOT EXISTS edge_video_analysis 
  DEFAULT CHARACTER SET utf8mb4 
  DEFAULT COLLATE utf8mb4_general_ci;

-- 执行建表脚本
source src/main/resources/db/init.sql

-- 创建索引
CREATE INDEX idx_sensor_lamp_time ON t_sensor_data(lamp_id, capture_time);
CREATE INDEX idx_alarm_record_lamp_time ON t_alarm_record(lamp_id, alarm_time);
CREATE INDEX idx_inference_lamp_time ON t_inference_record(lamp_id, inference_time);
```

### 3.2 启动依赖服务

#### MySQL (Windows)
```bash
net start MySQL80
```

#### Redis (Windows)
```bash
redis-server --requirepass 123456
```

#### YOLOv8 服务
```bash
cd yolov8-service

# 安装依赖
pip install -r requirements.txt

# 下载模型 (首次运行自动下载yolov8n.pt)
# 或手动下载到 yolov8n.pt

# 启动服务
python app.py
```

### 3.3 构建后端

```bash
# 编译打包
mvn clean package -DskipTests

# 运行
java -jar target/edge-video-analysis-1.0.0.jar
```

或直接通过IDE运行 `EdgeVideoApplication.java`

## 四、配置说明

### 4.1 环境变量

| 变量名 | 默认值 | 说明 |
|--------|--------|------|
| MYSQL_ROOT_PASSWORD | 123456 | MySQL密码 |
| MYSQL_DATABASE | edge_video_analysis | 数据库名 |
| REDIS_PASSWORD | 123456 | Redis密码 |
| YOLO_MODEL_PATH | yolov8n.pt | YOLO模型路径 |
| YOLO_CONFIDENCE_THRESHOLD | 0.5 | 置信度阈值 |
| BACKEND_PORT | 8080 | 后端端口 |

### 4.2 应用配置

编辑 `src/main/resources/application.yml`:

```yaml
# 数据库连接
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/edge_video_analysis
    username: root
    password: 123456
  redis:
    host: localhost
    port: 6379
    password: 123456

# AI服务地址
yolo:
  service:
    url: http://localhost:5000

# 视频线程池
video:
  thread:
    core-size: 4
    max-size: 7
    queue-capacity: 100
```

## 六、故障排查

### 6.1 常见问题

**Q: 容器启动后立即退出**
```bash
# 查看日志
docker-compose logs <service-name>
```

**Q: MySQL连接失败**
- 检查密码是否正确
- 确认MySQL服务已启动
- 检查防火墙3306端口

**Q: YOLOv8推理超时**
- 增加超时配置
- 检查模型文件是否存在
- CPU模式下推理较慢，可考虑使用GPU

**Q: WebSocket连接断开**
- 检查心跳间隔 (建议30秒)
- 确认Redis服务正常
- 查看网络代理设置

### 6.2 性能调优

```bash
# JVM参数优化 (Dockerfile)
JAVA_OPTS="-Xms1g -Xmx2g -XX:+UseG1GC"

# MySQL优化
max_connections=200
innodb_buffer_pool_size=1G

# Redis优化
maxmemory=512mb
maxmemory-policy=allkeys-lru
```

## 七、生产环境建议

1. **使用反向代理**: Nginx转发请求，配置SSL证书
2. **数据库备份**: 定期备份MySQL数据
3. **日志收集**: 使用ELK或Loki收集日志
4. **监控告警**: 接入Prometheus + Grafana
5. **高可用部署**: 多实例部署 + 负载均衡
6. **安全加固**: 
   - 修改默认密码
   - 限制API访问频率
   - 配置IP白名单
